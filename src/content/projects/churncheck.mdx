## Overview

**ChurnCheck** is an analytics platform developed during Oracle ONE Hackathon Latam for predicting customer churn. The system transforms raw customer data into actionable insights through machine learning, helping businesses identify at-risk customers and implement retention strategies.

**Live Demo**: Available during hackathon presentation
**Repository**: [github.com/sjlvanq/nc-oneiila-e14b](https://github.com/sjlvanq/nc-oneiila-e14b)

---

## Challenge Context

**Oracle ONE Hackathon - Latam** brought together multidisciplinary teams to solve intensive innovation challenges in 5 weeks. Our team of 9 members developed a complete customer churn prediction solution under extreme time pressure, competing against teams from across Latin America.

**Problem**: Customer churn represents significant financial impact for subscription-based businesses, with acquisition costs 5-7x higher than retention costs.

---

## Technical Solution

**ChurnCheck** implements a complete data-driven approach:

### Data Science Pipeline
- **Dataset Analysis**: Gym customer churn dataset with demographic, behavioral, and contract variables
- **Feature Engineering**: Created meaningful predictors from raw customer data
- **Model Selection**: Evaluated multiple algorithms, selected optimized Logistic Regression
- **Performance**: Achieved 92.8% accuracy, 84.5% recall, 85.2% F1-score

### Backend Architecture
- **Java 21 + Spring Boot 3**: RESTful API with JWT authentication
- **H2 Database**: In-memory database for rapid prototyping
- **Microservices**: Model deployed as independent FastAPI service
- **Security**: Stateless JWT-based authentication system

### Frontend Dashboard
- **Interactive Analytics**: Real-time churn probability visualization
- **Risk Classification**: Customer segmentation by risk level
- **Business Intelligence**: Trend analysis and predictive insights
- **Responsive Design**: Mobile-friendly interface with Chart.js

---

## Architecture

**Three-pill architecture ensuring scalability and maintainability:**

1. **Data Science Module**: Predictive analytics engine
2. **Backend API**: Business logic and integration layer  
3. **Frontend Dashboard**: User interface and visualization

**Communication Flow**: REST APIs with JSON format, enabling independent development and deployment of each component.
